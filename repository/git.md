### Git

#### アカウント作成
    ①公式サイトからgitforwindowsをダウンロードしてローカルにインストール(インストーラー起動後は基本全て「Next」でOK)
    ②コマンドプロンプトでユーザ名とメールアドレス登録
      git config --global user.name ユーザ名
      git config --global user.email メールアドレス
    ③以下のURLを開き、アカウントを作成
      https://github.co.jp

#### コマンド概要
    ・Fetch：他のユーザの反映をローカルに取り込む。ローカルのソースは更新されない。
      ⇒他のユーザがブランチを追加した場合に、そのブランチを見れるようにするなど。
    ・Pull：他のユーザの反映をローカルに取り込み、ローカルのソースも更新される。
    ・Add：ステージングに載せる。取り消したい場合はRevertを使う。
    ・Revert：コミット前のローカル修正ソースを元に戻す

#### クローン作成
    ①コンテキストメニューから「Git Clone…」を選択
    ②URLにmasterのURLを指定し、OKボタンを押下。この時にブランチを選択するとエラーとなり作成できない。
    ※ブランチを指定したい場合は以下の「ブランチの切替」を実行する。

#### フェッチ
    まずはフェッチし、リモートの最新の更新情報を取込む(ソースは取込まれない)

#### スイッチ（ブランチの切替）（TortoiseGit）
    ①ルートパスとなる親フォルダで右クリック
    ②「TortoiseGit」－「Switch/Checkout...」を選択
    ③「Switch To」欄のBranchで切り替えたいブランチを選択（「Option」欄は自動でセットされる）
    ※ローカルフォルダの資材も入れ替わる
    ※ローカルリポジトリ作成後は基本的にローカルリポジトリに対してスイッチすること。
    そうしないと、直接リモートに対してコミットしようとする→再度ローカルリポジトリを作成するかを聞かれたりもする。

#### ローカルブランチのチェックアウト
    ①ルートパスとなる親フォルダで右クリック
    ②「TortoiseGit」－「Switch/Checkout...」を選択
    ③「Switch To」欄のBranchで元なるブランチを選択し、「Option」の「Create New Branch」のチェックをオンにし、ブランチ名を設定
      例）元のブランチ：remotes/origin/feature/test
          作成するブランチ：feature/test_work
    ④以上で作成されるが、作成したローカルブランチをリモートにあげる時は、
    Pushすれば存在しないブランチだと自動的にリモートに作成してくれる・・・未確認

#### ブランチ作成
    ①GitLabのWeb画面でBranchesメニューを開く
    ②[New Branch]ボタンを押下
    ③[Branch Name]に作成するブランチの名称、[Create from]に作成元のブランチを指定して[Create Branch]ボタンを押下。
    ④ローカルのプロジェクトフォルダのルートでTortoiseGitの[Fetch]をしてから、[Switch/Checkout]で作成したブランチに切り替える。

#### コミット戻しの注意点
    一度コミットするとresetではうまく戻らなかった。revertで打ち消しコミットをすることになる。

#### プッシュのやり方
    TortoiseGitのメニューからPushでも良いが、「Git Sync…」メニューを開いて差分を確認してから、
    そのダイアログの中のPushボタンを使った方が良い。

#### ■ファイル削除の方法(ディレクトリ削除含む)
    ローカルで削除しコミット、プッシュが無難。
    ⇒リモート上で削除すると、それ以降のプッシュでエラーが発生し解消できなかったため、
    リモートリポジトリの削除とローカルのSourceTreeの再インストールするハメになった。
    ⇒ディレクトリをローカルでコピーし、古いディレクトリの削除とリネームした新しいディレクトリを
    コミット、プッシュしたが、問題なくリモートに反映された。

    ≪以下は古い記述≫
      エクスプローラーを右クリックし、「TortoiseGit」－「Delete」を選択する。
      ※TortoiseGitメニューから削除しないと、消失扱いになり、うまくいかない。
      もっと良いやり方があるのかもしれないが、消失扱いになったら、そのプロジェクトはリネームしてバックアップし、
      新たにクローンを落としてきて、やり直すのが無難。

#### マージリクエストの出し方
    ①GitLabのWeb画面で左欄のメニューからBranchesを開く。
      ※メニューが出ていない場合は上方のツールチップの左端に四角いマークがあるので、それをクリックしてメニューを出す。
    ②SourceBranchとTargetBranchを指定して、「Compare branches and continue」ボタンを押下。
    ③次の画面で、TitleとAssigner(自分)、Reviewer(レビューする者)を指定して「Create merge request」ボタンを押下。

#### コンフリクトの解消方法(リモート間：例えばリモートmasterとリモートbranchのマージコンフリクト)
    ①GitLabのWebのMerge Request画面で該当するマージリクエストを開く。
    ②コンフリクトの欄の[Resolve conflicts]を押下する。
    ③競合内容が表示されるため、以下のように修正する。
      ・自分と競合先の両方が必要な場合：[Edit Inline]ボタンを押し、画面上でソースコードを修正する。
      ・自分と競合先のどちらかを選択する場合：[Interactive Mode]ボタンを押し、[Use Ours]または[Use theirs]のどちらかを押下
    ④最後に一番下の[Commit to source branch]ボタンを押下する。→リモートbranchに修正が反映されてmasterにマージ可能になる。
    ⑤変更をローカルリポジトリにも反映するため、Pullを行う。→ローカルbranchに反映される。

#### 改行コード自動変換に注意
    TortoiseGitの[設定]-[Git]-[AutoCrLf]で設定・・・[Config source]欄のEffectiveがtrueだと自動変換される。
    →もしローカルを直してリモートに上げる方法（例えば実際はCrLfで誤ってLFになってしまっている場合）
    ①AutoCrLf設定をfalseにする。
    ②ローカルのファイルをいったんダミーの変更して、すぐにリバートするとローカルもCrLfからLFになる
      (既にLFになっていれば、この手順は不要)
    ③ファイルを開き、サクラエディタなどで改行コードをLFからCrLfに変更
    ④コミット＆プッシュ

#### リポジトリのURL確認方法
    ①エクスプローラでリポジトリ(チェックアウト済のフォルダ)を選択して右クリック
    ②[TortoiseGit]-[Show Reflog]を選択
    ③行を選択して右クリック
    ④[Copy to clipboard]-[Messages]を選択し、コピーされた情報からURLを取得する。
