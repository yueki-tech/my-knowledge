Java開発環境（Git）

■Java開発環境構築（Git）
①エクスプローラでワークスペース用のフォルダを作成する。（例：C:\workspace）
②GITのローカルリポジトリを格納するため、エクスプローラでリポジトリ用フォルダを作成する。（例：C:\git_repository）
③eclipseを起動し、右上の「パースペクティブを開く」をクリックする。
④パースペクティブから「Git」を選択し、「Git」パースペクティブを開く。
⑤「Git リポジトリ－」パースペクティブが開かれるので、画面に表示されている「Git リポジトリ－のクローン」リンクをクリックする。
⑥「リポジトリ－・ソースの選択」画面で「クローンURI」を選択し、「次へ」をクリックする。
⑦「ソース Git リポジトリ－」を選択する画面では以下を入力し、「次へ」をクリックする。
URI：（Gitのリモートリポジトリ－のURLを入力）
ホスト：（URIを入力すると自動設定される）
リポジトリ－パス：（URIを入力すると自動設定される）
プロトコル：http　※デフォルト
ポート：9090　※デフォルト
ユーザー：（リモートリポジトリ－に接続するためのユーザー名）
パスワード：（リモートリポジトリ－に接続するためのパスワード）
セキュア・ストアに保管：チェック不要
⑧「ブランチ選択」画面では「master」またはローカルにチェックアウトしたいブランチを選択し、「次へ」をクリックする。
⇒下記⑨の初期ブランチでブランチを選択すれば、ここは「master」を選択しても問題ないかもしれない。
⑨「ローカル宛先」画面では以下を設定、「完了」をクリック
ディレクトリー：上記②で作成したフォルダ
初期ブランチ：「master」またはローカルにチェックアウトしたいブランチを選択
サブモジュールのクローン：チェック不要
リモート名：origin
クローン終了後、すべての既存Eclipseプロジェクトをインポート：チェック不要
ワーキング・セットにプロジェクトを追加：チェック不要
⑩ローカルリポジトリが作成されているので、eclipseの右上のパースペクティブから「Java」（STSの場合は「Spring」）をクリックする。
⑪ファイルメニューから「インポート」を選択する。
⑫「選択」画面で「Git＞Gitからプロジェクト」を選択し、「次へ」をクリックする。
⑬「リポジトリ－・ソースの選択」画面で「既存ローカル・リポジトリ－」を選択し、「次へ」をクリックする。
⑭「Gitリポジトリ－の選択」画面で⑨で作成したローカルリポジトリ－を選択し、「次へ」をクリックする。
⑮「プロジェクトをインポートに使用するウィザードの設定」画面で、「既存プロジェクトのインポート」を選択の上、「作業ツリー」を選択し、「次へ」をクリックする。
⑯「プロジェクトのインポート」画面で、対象のプロジェクトを選択し、「ネストされたプロジェクトの検索」にチェックを付け、「完了」をクリックする。
※「ワーキング・セット」は未チェックで良い。
⑰プロジェクトが作成されるので、プロジェクトを右クリックし、コンテキストメニューから「Maven＞プロジェクトの更新」を選択する。

※STSの場合は、このまま実行ボタンを押下し、「Spring Bootアプロケーション」を選択して実行すれば、サーバが起動してWebサイトにアクセスできるようになる。


■マージ ※ブランチをmasterにマージ
①masterブランチを取得する。
masterがローカルに無ければチェックアウト。有れば最新をプルする。
②ブランチをmasterにマージする。
マージでは、マージされる方のブランチを表示して作業する。
ブランチのプロジェクトをチェックアウトしている状態であれば、[チーム]＞[切り替え]＞[master]を選択し、masterブランチへ切り替えた上で、[チーム]＞[マージ]を選択する。
次にマージするブランチを選択し、[マージ]ボタンを押下する。
③masterをプッシュする。
マージしたmasterブランチのプロジェクトを右クリックし、[チーム]＞[アップストリームへプッシュ]を選択する。


■タグ付け
①eclipseで「Gitリポジトリ－」パースペクティブを開き、masterの「タグ」カテゴリを右クリックし、「タグの作成」を選択する。
②タグ名、タグ・メッセージを入力し、「タグを作成してプッシュ開始」ボタンを押下する。
③プッシュするタグを選択するダイアログが表示されるので、今回作成したタグが選択されていることを確認し、「次へ」ボタンを押下する。
